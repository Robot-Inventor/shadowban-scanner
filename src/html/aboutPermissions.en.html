Details

Within a few days, a Shadowban Scanner will display a dialog in your browser asking you to approve additional permissions. Please approve the permissions.

The reason why additional permissions are required is to accommodate changes in Twitter specifications. Details are explained in the following explanation.

Why do it need new permissions?

In order to accommodate Twitter (X) specification changes, the next update of Shadowban Scanner will add required permissions. Until now, the following host permissions were used.

  • https://twitter.com/*
  • https://mobile.twitter.com/*
  • https://tweetdeck.twitter.com/*

This means that Shadowban Scanner will be loaded and work with the above URLs. These are the URLs for Twitter and TweetDeck.

However, due to recent changes in Twitter specifications, the URL of TweetDeck (X Pro) has changed. Shadowban Scanner can only work with the above URLs, so it no longer works with TweetDeck.

Therefore, in the next update, I will change the host privileges as follows

  • https://*.twitter.com/*
  • https://*.x.com/*

This means that Shadowban Scanner will have access to all Twitter URLs, including the new TweetDeck URL. x.com is included to accommodate Twitter's future URL changes.

After changing the permissions, Shadowban Scanner will still only be enabled on Twitter and TweetDeck. It's no longer enabled on Twitter's help page, etc., so it's essentially no different than before. Things that stopped working due to specification changes will simply return to normal.

What happens if you don't approve the permissions?

Approving permissions is not required, but due to browser specifications, failure to approve permissions will prevent you from updating Shadowban Scanner.

This is a browser specification and there is nothing I can do about it. I have looked for other ways, but this is the only way I could find.

Is it not possible to ask for additional permissions only for those who want to use it in TweetDeck?

No I can't, to support TweetDeck I would have to ask for permissions with all Shadowban Scanner users.

To be precise, we used to be able to do so, but due to changes in browser specifications (introduction of Manifest V3), it is no longer possible. I hope you understand this.
